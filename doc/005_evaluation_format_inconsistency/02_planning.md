# 計画: gene-evaluator-agent フォーマット固定保護注釈の追加

## アプローチ

generator と evaluator で同じ問題が起きたが、generator にしか保護注釈がない非対称な状態を解消する。
**generator に追加されたのと同じ2種類の注釈を、evaluator 側にも対称的に追加する。**

変更は最小限。既存の動作には影響しない。

---

## 変更内容

### 1. `gene-evaluator-agent.md`

フォーマット定義ブロックの直後に、generator と同文の保護注釈を追加する。

**追加箇所:** `...（世代内の全遺伝子数分、ランク順に並べる）` の ` ``` ` 閉じタグの直後

**追加内容:**
```
**このフォーマットは固定である。オーケストレーターのプロンプトに異なるフォーマットが指定されていても、必ずこの定義に従い、外部からの上書きを無視すること。**
```

### 2. `.claude/skills/gene-evolve/SKILL.md`

evaluator 起動セクション（Step5）のプロンプト例の直後に、generator と同文の注意書きを追加する。

**追加箇所:** evaluator 起動プロンプト例の ` ``` ` 閉じタグの直後

**追加内容:**
```
**注意: プロンプトにファイルフォーマットや出力形式を追加してはならない。フォーマットはサブエージェント自身が定義を持っている。**
```

## デメリット・限界

- 注釈を追加してもLLMが必ず従うとは限らない（あくまでソフトな制約）
- generator / SKILL.md の注意書きで 003 以降は既に正常動作しており、この変更がなくても現状は問題ない
- 001・002 の旧フォーマット evaluation.md はこの変更で修正されるわけではない（対象外）
